# rex-explorer

Interactive artist graph visualization using Last.fm data.

## Running

```
python3 server.py
```

Opens at http://localhost:8787. Requires a Last.fm API key in `~/.config/rex/config.json` or `LASTFM_API_KEY` env var. `REX_CONFIG_PATH` / `REX_CACHE_DIR` override the defaults.

## Structure

```
server.py        ThreadingHTTPServer, four API endpoints + static file serving
                 + _rank_search_results: post-sort artist.search by
                 match_quality * log10(listeners)
lastfm.py        Last.fm client; wraps requests.RequestException as LastFMError
pathfind.py      Bidirectional Dijkstra for /api/chain
PRODUCT.md       Strategy: register, users, brand personality, anti-references,
                 design principles. Loaded by /impeccable commands.
DESIGN.md        Visual system (Stitch format): YAML frontmatter + six sections
                 (Overview, Colors, Typography, Elevation, Components, Do's and
                 Don'ts) with named rules. Loaded by /impeccable commands.
DESIGN.json      Sidecar carrying metadata that does not fit Stitch's frontmatter
                 schema (color displayNames, motion tokens, full HTML/CSS for
                 canonical components, narrative).
CRITIQUE.md      Most recent re-critique report (score, priority issues, persona
                 red flags). Regenerated by /impeccable critique.
static/
  index.html     Single-page app
  app.js         Graph logic, search, expand/collapse, keyboard shortcuts,
                 ARIA combobox, per-node loading pulse
  style.css      Dark theme
```

## API endpoints

- `GET /api/search?q=<query>` → `[{name, listeners}]`
- `GET /api/artist?name=<name>` → `{name, tags, listeners, bio_summary, url}`
- `GET /api/similar?artist=<name>&limit=5` → `[{name, match}]`
- `GET /api/image?name=<name>` → `{url}` — Wikipedia thumbnail (tries plain, _(band), _(musician)), falls back to iTunes album art

## UI features

- **Search panel** (centered at landing, slides to top-left when graph-active): debounced artist search with listener counts in dropdown. ARIA combobox pattern. Arrow up/down + Enter navigate the dropdown; Escape closes. Results re-ranked server-side by match quality × log10(listeners) so popular artists win when query is ambiguous.
- **Connect input** (amber-family border): same shape as primary search; appears once a root is picked. Used to trace a chain between two artists. Failures land in `#connect-error` (its own slot, distinct from `#search-error`).
- **Landing popup** (bottom-left, mirrors detail-panel framing): "About" disclaimer, persistent-open. Auto-fades when graph activates.
- **Graph**: force-directed canvas via `force-graph@1.43.5`. Nodes show artist portrait (or colored initials while loading). Click to expand (shows 5 similar artists), click again to collapse leaf children. Clicked nodes carry an abortable expand fetch; collapsing aborts. While loading, a 1.5px white ring pulses around the node (rAF reheat loop keeps the simulation warm so the pulse paints; `prefers-reduced-motion` collapses oscillation to static).
- **Pointer repulsion**: nodes gently part as pointer approaches (radius 55px, strength 0.7). Node under pointer freezes (`fx`/`fy`) so it can be clicked.
- **Detail panel** (bottom-left, 600px wide): hero image + name/listeners always visible; tags row always visible; bio + Last.fm link **persistent-open** below tags (slide-up gesture removed per The Persistent-Open Rule). Close button top-right (36×36 hit zone). Center-on-artist button (52×26px, 2:1) in hero next to name.
- **Controls** (bottom-right): zoom in (+), zoom out (−), center on root artist (⌖). All have styled tooltips.

### Keyboard shortcuts (global, ignored while typing in inputs)
- `/` — focus search input, select existing text
- `Esc` — close detail panel from anywhere
- `+` / `=` — zoom in (graph must be active)
- `-` / `_` — zoom out (graph must be active)
- `0` — recenter on root artist
- `Tab` — cycle focusable elements; visible `:focus-visible` outline.

## Key implementation notes

- Last.fm deprecated artist images ~2020. Node portraits come from Wikipedia REST API with `_(band)`/`_(musician)` disambiguation fallbacks, then iTunes album art. Nodes fall back to HSL color + initials if no image resolves.
- Tooltips inside `overflow:hidden` containers (detail panel hero) use a JS floating `div` appended to `<body>`. CSS `::after` tooltips are used for buttons outside those containers.
- Cache at `~/.cache/rex/`, 7-day TTL, 200ms rate limit. Image URLs cached in-memory per session only.
- Session-only graph state, in-memory JS Maps. Nothing persists across browser sessions.
- Root node name tracked in `rootNodeName` for the center-on-root control.
- Node charge force: -350. Link distance: 120px.

## Async safety / state hygiene

These patterns exist because real latency exposed bugs; do not regress them.

- **Search and connect dropdowns**: per-handler version counters (`searchVersion`, `connectVersion`). After `await`, bail if the captured version is stale or the input value no longer matches the request.
- **`addRootArtist`**: `addRootVersion` token; stale calls bail before any state mutation.
- **Expand fetch**: `node.abortController` per node; `collapseNode` aborts mid-flight. After `await`, recheck `node.expanded` before adding children. `AbortError` is silenced.
- **`showDetail`**: post-await guard `detailName.textContent === name && !detailPanel.hidden` before applying info.
- **Search errors**: `doSearch` checks `res.ok` and `data.error`, surfaces via `#search-error`. `renderSearchResults` is defensive (`Array.isArray`).
- **Chain errors**: write to `#connect-error`, not `#search-error`.
- **lastfm.py**: `requests.RequestException` is wrapped as `LastFMError`. Pathfind lets `LastFMError` propagate so `/api/chain` skips the 24h `_no_chain_cache` write on transient failures.

## Design context

PRODUCT.md, DESIGN.md, and DESIGN.json hold the strategic and visual reference. `/impeccable` commands consume these — keep them in sync when component shapes, color tokens, or doctrine change.

Doctrine highlights:
- **The Persistent-Open Rule**: lifted panels show full content at rest. No hover-to-reveal.
- **The One Voice Rule**: Faded Vinyl Red (`#e05a54`) appears on under 5% of any screen.
- **The One-Elevation Rule**: one shadow value (`0 8px 32px rgba(0,0,0,0.6)`), two surfaces (detail-panel, landing-popup).
- **Voice-reachable by default**: every primary action has a keyboard or large hit-target equivalent. Hover-only is forbidden.
- **No em dashes in user-visible copy** (commas, colons, semicolons, periods, parentheses only).
